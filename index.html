1. INSTALLATION

The project internally uses Text Mesh Pro and Input System.

TEXTMESH PRO IN THE PROJECT

The TextMesh Pro package is included in Unity Editor. It is not necessary to install it. You can follow the simple guide at the link to insert it into the project (TextMesh Pro).

There may be an issue where you do not see the text on buttons or other parts, due to the text not having an assigned font. Just go to the text and set a base or custom font.

INPUT SYSTEM

At the beginning of the project, there is a basic player that can move freely and access the Assembly Station via a button. The player is moved thanks to the Unity Input System and contains a script that manages the Assembly Station resources.

If your project already has a player, you can use your Input System and add the script RESOURCES COMPONENT PLAYER to the components. If you want to use mine, you need to install the INPUT SYSTEM.

If an Input System is already present, just modify the script PLAYER CONTROLLER to integrate it.

3. INSTANTIATE PREFABS

Once the materials are separated, go to the Prefab Clone folder: Assets > AssemblyStation > Prefab Clone.

The Assembly Station instantiates the prefab at a specific point called Assembly Point.

That point refers to the pivot of the prefab's parent, which contains all the “pieces” that make up the gameobject.

When we instantiate the prefab, the pivot point corresponds to the Assembly Point. To ensure that the prefab does not float or intersect with the ground, you need to move the parent's pivot. (Starting from the FBX)

In this case, as you can see, the pivot is slightly shifted upwards. So we press Unpack Completely.

Select everything except the parent (in this case Mech Rework) and move everything out of the parent.

Select the parent (Mech Rework) and move the pivot.

To move the pivot, just click on the green arrow and move it downwards. I recommend activating the 2D view for a better view.

As you can see, the model will not move in space but only the pivot.

In this case, the Mecha is aligned at the bottom to make it appear resting on the ground. So I moved the pivot slightly downwards. This depends on you based on how you want to display the model.

You can also position the model in the air (for example, if it is a weapon). In this case, the pivot is very low, so the model remains suspended.

Creating the Prefab

Once done, select all the children and put them back inside the parent as before. Then, drag the parent into the (Prefab clone) folder and a Prefab will be created automatically.

4 - ADD DESCRIPTIONS AND IMAGES TO PREFABS

Once you have added all the desired prefabs and moved the pivots, you need to add a description and an image to each child that contains MeshRenderer or SkinnedMeshRenderer, while the parent needs to be assigned a title.

All this is done automatically by the asset (depending on the number of prefabs in the folder, it will take more or less time).

Just click on the button (at the top) Tools > Add Name and Descriptions To Prefab select the prefab clone folder and a description, a name, and an image will be automatically assigned to each child and a title to the parent.

Do not insert anything inside the field

IMAGES

Each "piece" will have an associated image that will be displayed in the preview.

The image is generated automatically when we press the button Tools > Add Name and Descriptions To Prefab
If you want to insert a personal image, just go to the Piece Description and select the desired image, replacing the one generated by the system.
Functionality (Editor Mode): when clicking on Add Name and Descriptions To Prefab, a camera is generated in front of the piece and a photo is taken, this photo is saved in the folder Assets > AssemblyStation > Screenshot > and is associated with the child. For obvious reasons, the image may be compromised by the size of the piece and an incorrect angle. For this reason, it is recommended to insert a personal image if this problem occurs.

6 - TAGS/CATEGORIES/GROUPS OF PIECES

Once all the texts and images have been inserted, you can proceed with creating the TAGS and GROUPS.

Initially, each piece of the prefab will not have a tag and will therefore be visible in the panel in a single list.

If you want to divide the pieces by category, just create a tag and assign it to each piece for each category. You can select multiple tags at the same time by holding down the CTRL key and selecting the various Pieces, otherwise by pressing the CTRL+SHIFT key.

By doing so, the pieces will be divided based on the tag you assigned.

ATTENTION!!!

Pay attention to this part

GROUPS

Each piece that contains a SkinnedMeshRenderer or a MeshRenderer will be displayed in the panel. This means that, in the presence of models with hundreds of pieces, the resulting list will be very long, making it difficult for the user to quickly find the desired piece.

7- RESOURCES

The Resources section is more complex and requires programming knowledge. For obvious reasons, I cannot know which and how the resources are managed in your game, so I have added some random ones. Iron, Plastic, Rubber. By default, the assembly station uses a resource system, where each piece has a cost and is deducted from the player's inventory.

The player's resources are located in the folder Assets > AssemblyStation > ScriptableObject > Resources Player. this is assigned to the test Player through the script(ResourcesComponentPLayer) that you can find in PLayer Test in the Hierarchy

Here you will assign the quantity of resources, while in the script(PlayerResources) you can manage the logic.

As seen in chapter 4 this is the section where you can add the various texts, images, etc. in the single piece. As you can see, there are 2 sections we haven't talked about RESOURCES VISIBLE and RESOURCES. Here you can enter the costs for each piece and make the resources visible or not. example:

ATTENTION!!!

For the pieces that have a Group, the same rule always applies, the first in the list is taken. So the resources will need to be changed there.

In the case of pieces with a Group, remember that the resources must always be changed in the first piece of the Group, as the data related to image, description, and cost are taken from there.
